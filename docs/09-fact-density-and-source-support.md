# 09. Fact Density and Source Support

Fact Density means how much useful, specific, and verifiable information appears inside a piece of content.

High Fact Density does not mean stuffing the page with random facts. It means each section contains clear, useful, and checkable information.

## Fact Density Pattern

```text
Definition → Explanation → Example → Mistake → Checklist → Verification
```

## Low vs High Fact Density

| Low Fact Density | High Fact Density |
|---|---|
| General advice | Specific action steps |
| Vague claims | Checkable claims |
| Long paragraphs with little value | Short sections with clear meaning |
| No examples | Real or realistic examples |
| No verification | Source, tool, checklist, or method |

## Source Support

Use source support when the content includes:

- Standards
- Data
- Legal or policy claims
- Security guidance
- Technical documentation
- Tool-specific instructions
- Claims that may change over time

## Verification Guidance

Examples:

```text
You can check this in Google Search Console.
You can verify this by inspecting the canonical tag.
You can test this with a rich result validator.
You can confirm this by opening the live URL and checking the source code.
```

## Checklist

- [ ] Does each H2 contain a clear answer?
- [ ] Does each section include specific details?
- [ ] Are vague claims removed?
- [ ] Are examples added where useful?
- [ ] Are technical claims verifiable?
- [ ] Are tools or methods mentioned where appropriate?
- [ ] Are outdated claims reviewed?
- [ ] Are sources added when needed?
