# 20. Cannibalization and SERP Fit Layer

Cannibalization happens when multiple pages compete for the same keyword or search intent.

SERP Fit means the page type matches what users and search engines expect for that query.

## Common Cannibalization Cases

- Blog post competes with service page
- Category page competes with product page
- Two blog posts target the same keyword
- Multi-domain websites target the same topic without clear role
- Author page overlaps with service page

## Primary Page vs Supporting Page

For each keyword group, define:

```text
Primary page: the page that should rank
Supporting pages: pages that explain subtopics and link to the primary page
```

## Example

Keyword:

```text
dịch vụ SEO tổng thể
```

Primary page:

```text
/dich-vu-seo-tong-the/
```

Supporting pages:

```text
/seo-tong-the-la-gi/
/seo-tong-the-va-seo-tu-khoa/
/quy-trinh-seo-tong-the/
```

Supporting pages should link back to the primary page and avoid acting like the main service page.

## SERP Fit Checklist

- [ ] Does the page type match the keyword intent?
- [ ] Is this the correct primary page?
- [ ] Are supporting pages clearly different?
- [ ] Do supporting pages link to the primary page?
- [ ] Are titles too similar?
- [ ] Are H1s too similar?
- [ ] Is the CTA appropriate for the intent?
- [ ] Is canonical/index status correct?

## Cannibalization Fix Options

- Merge two weak pages
- Change the target keyword
- Rebuild one page as a supporting article
- Strengthen the primary page
- Add internal links to the primary page
- Deindex low-value duplicate pages when appropriate
- Update title/H1 to clarify unique intent
