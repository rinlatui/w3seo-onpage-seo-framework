# 17. UX and Readability Layer

Onpage SEO is not only keyword optimization. Content must be easy to read, scan, understand, and act on.

## Readability Principles

- Use short paragraphs.
- Use clear headings.
- Use tables for comparison.
- Use bullet lists for scanning.
- Use examples for understanding.
- Avoid long walls of text.
- Avoid unclear jargon.
- Make mobile reading comfortable.

## Above-the-Fold Clarity

The first screen should quickly answer:

- What is this page about?
- Who is it for?
- What problem does it solve?
- What should the user do next?

## Section Design

Each section should have:

```text
Clear H2 → Short answer → Explanation → Example/checklist → Next step
```

## Mobile Readability Checklist

- [ ] Paragraphs are short
- [ ] H2 sections are clear
- [ ] Tables are not too wide
- [ ] CTA is visible
- [ ] Images do not break layout
- [ ] Text is not too dense

## Content Formatting Checklist

- [ ] Use H2 for main ideas
- [ ] Use H3 for supporting ideas
- [ ] Use lists for steps
- [ ] Use tables for comparison
- [ ] Add visual breaks
- [ ] Avoid repeating the same idea
